# 项目并发处理分析报告

## 一、概述

本项目（Learn Work Agent）是一个基于 Spring Boot 的智能学工系统，采用多种技术手段应对高并发场景。系统通过异步处理、消息队列、缓存、响应式编程等技术，实现了高效稳定的并发处理能力。

---

## 二、并发处理技术方案

### 1. 异步任务处理

#### 1.1 线程池配置

**配置文件**: `infrastructure/config/AsyncConfig.java`

```java
@Configuration
@EnableAsync
public class AsyncConfig {
    @Bean(name = "taskExecutor")
    public Executor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);        // 核心线程数
        executor.setMaxPoolSize(10);        // 最大线程数
        executor.setQueueCapacity(100);     // 队列容量
        executor.setThreadNamePrefix("async-");
        executor.initialize();
        return executor;
    }
}
```

**线程池参数说明**:
| 参数 | 值 | 说明 |
|------|-----|------|
| 核心线程数 | 5 | 常驻线程数量 |
| 最大线程数 | 10 | 紧急情况可扩展到的最大线程 |
| 队列容量 | 100 | 等待队列大小 |
| 线程前缀 | async- | 便于日志追踪 |

```java
@Transactional
public void preCheckMaterialsAsync(Long applicationId) {
    log.info("异步材料预审开始，申请ID: {}", applicationId);
    AwardApplication application = getApplicationById(applicationId);
    Map<String, Object> result = checkMaterialsComplete(application);
```

**场景二：咨询问题异步处理**

文件: `domain/consultation/service/ConsultationAgentService.java`

```java
@Async
public void processQuestionAsync(Long questionId) {
    ConsultationQuestion question = consultationQuestionRepository.findById(questionId)
            .orElseThrow(() -> new BusinessException(ResultCode.PARAM_ERROR, "问题不存在"));

    String cachedAnswer = getCachedAnswer(question);
    if (cachedAnswer != null) {
        self.updateQuestionAnswer(question, cachedAnswer, "AI");
        return;
    }

    difyChatService.chatStream(prompt, fileUrls, null, String.valueOf(question.getUserId()), convId -> {
            question.setConversationId(convId);
            consultationQuestionRepository.save(question);
        })
        .collectList()
        .subscribe(answerList -> {
            String answer = String.join("", answerList);
            cacheAnswer(question, answer);
            self.updateQuestionAnswer(question, answer, "AI");
        });
}
```

### 2. 响应式流处理

#### 2.1 WebFlux 异步非阻塞通信

**配置文件**: `infrastructure/config/WebClientConfig.java`

```java
@Configuration
public class WebClientConfig {
    @Bean
    public WebClient.Builder webClientBuilder() {
        HttpClient httpClient = HttpClient.create()
                .responseTimeout(Duration.ofMinutes(2))
                .option(ChannelOption.CONNECT_TIMEOUT_MILLIS, 30000);

        return WebClient.builder()
                .clientConnector(new ReactorClientHttpConnector(httpClient))
                .codecs(configurer -> configurer.defaultCodecs().maxInMemorySize(10 * 1024 * 1024))
                .defaultHeader("Accept", "application/json");
    }
}
```

#### 2.2 Dify 聊天流式响应

**文件**: `infrastructure/external/dify/DifyChatService.java`

```java
public Flux<String> chatStream(String query, List<String> fileUrls, String conversationId, String user) {
    Map<String, Object> requestBody = buildChatRequestBody(query, fileUrls, conversationId, user);

    return webClient.post()
            .uri(baseUrl + "/v1/chat-messages")
            .header("Authorization", "Bearer " + chatApiKey)
            .contentType(MediaType.APPLICATION_JSON)
            .bodyValue(requestBody)
            .retrieve()
            .bodyToFlux(String.class)
            .map(chunk -> {
                // 解析SSE流式数据
                JsonNode jsonNode = objectMapper.readTree(jsonStr);
                JsonNode answerNode = jsonNode.get("answer");
                return answerNode.asText();
            })
            .filter(chunk -> !chunk.isEmpty());
}
```

**关键特性**:
- 使用 `Flux<String>` 实现 Server-Sent Events (SSE) 流式响应
- 非阻塞式 HTTP 调用
- 支持 conversation_id 捕获用于多轮对话

```java
public class RabbitMQConfig {

        SimpleRabbitListenerContainerFactory factory = new SimpleRabbitListenerContainerFactory();
```

### 4. WebSocket 实时通信

#### 4.1 WebSocket 配置

```java
@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    @Override
    public void configureMessageBroker(MessageBrokerRegistry config) {
        config.enableSimpleBroker("/topic", "/queue");  // 消息代理前缀
        config.setApplicationDestinationPrefixes("/app"); // 应用前缀
        config.setUserDestinationPrefix("/user");         // 用户前缀（点对点）

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws")
                .setAllowedOriginPatterns("*")
                .addInterceptors(webSocketAuthInterceptor)
                .withSockJS();
    }
}
```

#### 4.2 用户身份拦截

**文件**: `infrastructure/config/WebSocketChannelInterceptor.java`

```java
@Component
public class WebSocketChannelInterceptor implements ChannelInterceptor {
    @Override
    public Message<?> preSend(Message<?> message, MessageChannel channel) {
        StompHeaderAccessor accessor = MessageHeaderAccessor.getAccessor(message, StompHeaderAccessor.class);

        if (accessor != null && StompCommand.CONNECT.equals(accessor.getCommand())) {
            Object userId = Objects.requireNonNull(accessor.getSessionAttributes()).get("userId");
            accessor.setUser(new StompPrincipal(userId.toString()));
        }
        return message;
    }
}
```

**应用场景**:
- 审批通知实时推送
- 咨询问题回答实时推送
- 系统消息实时广播

### 5. Redis 分布式缓存

#### 5.1 Redis 配置

**文件**: `infrastructure/config/RedisConfig.java`

```java
@Configuration
@EnableCaching
public class RedisConfig {
    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory connectionFactory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(connectionFactory);

        ObjectMapper mapper = new ObjectMapper();
        mapper.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        mapper.activateDefaultTyping(mapper.getPolymorphicTypeValidator(), ObjectMapper.DefaultTyping.NON_FINAL);
        Jackson2JsonRedisSerializer<Object> serializer = new Jackson2JsonRedisSerializer<>(mapper, Object.class);

        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(serializer);
        template.afterPropertiesSet();
        return template;
    }

    @Bean
    public CacheManager cacheManager(RedisConnectionFactory connectionFactory) {
        RedisCacheConfiguration config = RedisCacheConfiguration.defaultCacheConfig()
                .entryTtl(Duration.ofMinutes(30))  // 默认过期时间30分钟
                .serializeValuesWith(RedisSerializationContext.SerializationPair.fromSerializer(serializer));
        return RedisCacheManager.builder(connectionFactory).cacheDefaults(config).build();
    }
}
```

#### 5.2 缓存服务

**文件**: `infrastructure/service/CacheService.java`

```java
@Service
public class CacheService {
    private final RedisTemplate<String, Object> redisTemplate;

    public void set(String key, Object value, long timeout, TimeUnit unit) {
        redisTemplate.opsForValue().set(key, value, timeout, unit);
    }

    public Object get(String key) {
        return redisTemplate.opsForValue().get(key);
    }
}
```

#### 5.3 咨询答案缓存

**文件**: `domain/consultation/service/ConsultationAgentService.java`

```java
private String getCachedAnswer(ConsultationQuestion question) {
    String cacheKey = "consultation:answer:" + question.getQuestionText().hashCode();
    Object cached = cacheService.get(cacheKey);
    return cached != null ? cached.toString() : null;
}

private void cacheAnswer(ConsultationQuestion question, String answer) {
    String cacheKey = "consultation:answer:" + question.getQuestionText().hashCode();
    cacheService.set(cacheKey, answer, 30, TimeUnit.MINUTES);
}
```

**缓存策略**:
| 缓存内容 | Key 格式 | 过期时间 |
|---------|---------|---------|
| 咨询答案 | `consultation:answer:{问题文本hashCode}` | 30分钟 |

## 四、技术总结

| 场景 | 技术方案 | 并发能力 | 优势 |
|------|---------|---------|------|
| **异步任务** | @Async + ThreadPoolTaskExecutor | 5-10 线程 | 非阻塞执行，提升吞吐量 |
| **流式响应** | WebFlux Flux (Reactor) | 无限制 | 背压机制，内存高效 |
| **消息队列** | RabbitMQ | 3-10 消费者 | 削峰填谷，解耦服务 |
| **实时通信** | WebSocket + STOMP | 无限制 | 实时推送，双向通信 |
| **数据缓存** | Redis | 无限制 | 减少数据库压力 |
| **分页查询** | Spring Data JPA | 无限制 | 内存友好 |
| **事务管理** | @Transactional | - | 数据一致性保证 |

## 五、关键设计模式

### 1. 异步解耦模式
```
事务提交 → 异步任务触发 → 后台处理 → 结果存储
```
适用场景：材料预审、答案缓存等非实时性任务

### 2. 消息队列模式
```
请求 → 消息队列 → 消费者池 → 多渠道发送
```
适用场景：审批通知、邮件发送等需要多消费者处理的场景

### 3. 缓存优先模式
```
请求 → 缓存查询 → 命中? → 返回 | 调用服务 → 结果缓存 → 返回
```
适用场景：高频问题的智能问答

### 4. 响应式流模式
```
请求 → 非阻塞调用 → 流式响应 → 实时推送
```
适用场景：AI 对话等需要实时输出的场景

## 六、总结

本项目通过多层次的并发处理技术，构建了一个高并发、高可用的智能学工系统：

1. **异步化**：使用 @Async 和线程池处理非实时任务
2. **响应式**：采用 WebFlux 实现非阻塞流式处理
3. **消息化**：RabbitMQ 解耦服务，实现弹性扩展
4. **实时化**：WebSocket 提供即时通信能力
5. **缓存化**：Redis 减少重复计算，提升响应速度
6. **安全化**：事务管理确保数据一致性

这些技术的有机结合，使系统能够有效应对并发请求，提供流畅的用户体验。
